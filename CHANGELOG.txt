FORMATO DE VERSIÓN: MAYOR.MENOR.PARCHE
  MAYOR  → Cambios incompatibles o rediseños completos
  MENOR  → Nuevas funcionalidades compatibles hacia atrás
  PARCHE → Correcciones de errores y ajustes menores

TIPOS DE CAMBIO:
  [ADD]    → Nueva funcionalidad añadida
  [FIX]    → Corrección de error o bug
  [CHANGE] → Modificación de funcionalidad existente
  [REMOVE] → Eliminación de funcionalidad
  [STYLE]  → Cambios de diseño o estilos visuales
  [DOCS]   → Cambios en documentación
  [PERF]   → Mejoras de rendimiento
  [SCALE]  → Preparación para escalabilidad

================================================================================
  INSTRUCCIONES PARA REGISTRAR UN NUEVO CAMBIO
================================================================================

Al realizar cualquier modificación al proyecto:

  1. Abre este archivo CHANGELOG.txt
  2. Ubica la sección correspondiente o crea una nueva versión:
       ========================================
         VERSIÓN X.Y.Z — Descripción breve
         Fecha: YYYY-MM-DD
         Autor: Tu nombre
       ========================================
  3. Agrega el cambio con el prefijo correcto: [ADD], [FIX], [CHANGE], etc.
  4. Actualiza el número de versión en package.json
  5. Haz commit con el mensaje: "chore: bump version to X.Y.Z"

  EJEMPLO:
       [FIX] Corregido bug donde el teclado numérico no respondía en iOS Safari
       [ADD] Agregado modo oscuro togglable desde el header
       [CHANGE] MAX_ATTEMPTS cambiado de 5 a 7 por defecto
